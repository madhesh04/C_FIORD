Bootstrapping SAPUI5

Lesson 2

To use SAPUI5 you must load and initialize the library. This is done via a bootstrap <script> tag that loads sap-ui-core.js — the standard bootstrap file (jQuery + jquery-ui-position + minimal sap.ui.core).

<script
  id="sap-ui-bootstrap"
  src="resources/sap-ui-core.js"
  data-sap-ui-theme="sap_fiori_3"
  data-sap-ui-libs="sap.m"
  data-sap-ui-compatVersion="edge"
></script>
📍
Local load — relative path from an SAP Web server.
🌐
CDN load — versioned URL (e.g. ui5.sap.com/1.96.16/...) for a fixed version, or the generic/default URL for testing only.

Key Bootstrap Configuration Options

AttributePurpose
data-sap-ui-themeTheme to use (e.g. sap_fiori_3, sap_horizon)
data-sap-ui-libsList of libraries to load initially (e.g. sap.m)
data-sap-ui-compatVersionMust be edge — enables the latest features; other values deprecated
data-sap-ui-resourcerootsMaps a module Id prefix to an app-specific folder (see Unit 2)
data-sap-ui-asyncLoads modules & library-preload files asynchronously
data-sap-ui-onInitModule (prefixed with module:) to run after initialization
⚠️
Exam tip: compatVersion must always be set to edge — this is required to enable complex/expression binding syntax used throughout later units.

Modularization — sap.ui.define

Lesson 4

Instead of one large bundle, SAPUI5 apps are split into modules loaded on demand. A module is defined with sap.ui.define, whose dependencies array lists the module names it needs (control class dots become slashes, e.g. sap.m.Text → sap/m/Text).

↑ A module's factory function receives one parameter per dependency, in the same order

sap.ui.define(["sap/m/Text"], function (Text) {
  "use strict";
  new Text({ text: "Hello World" }).placeAt("content");
});

Loading Configuration Options

OptionPurpose
resourcerootsMaps a module Id prefix (e.g. sap.training.exc) to a folder — lets SAPUI5 find app-specific modules
asyncLoads single modules & library-preload files asynchronously
onInitRuns a module:-prefixed module after framework init
💡
"use strict" (ECMAScript 5) inside the factory function catches coding problems early — e.g. using variables before declaration. Good practice in every module.
🎨

MVC: XML Views & Controllers

Lessons 6–7

MVC separates presentation from interaction. SAPUI5 supports XML, JSON, and Typed views — XML views are recommended because they cleanly separate UI definition (view) from logic (controller).

↑ XML view, JSON view, Typed view — XML is the course standard

<mvc:View
  xmlns:mvc="sap.ui.core.mvc"
  xmlns="sap.m"
  controllerName="sap.training.exc.controller.App">

  <Button text="Say Hello" press=".onSayHello"/>

</mvc:View>

📛 View Naming

File App.view.xml in the view folder + resourceroots prefix → module name sap.training.exc.view.App.

🏗️ Instantiation

XMLView.create({id, viewName}) loads asynchronously and returns a Promise resolving with the view instance.

🎮 controllerName

Attribute on <View> resolves to <prefix>.controller.<Name>.controller.js.

⚡ Event Handlers

A dot-prefixed attribute value (e.g. press=".onSayHello") calls that method on the controller.

sap.ui.define([
  "sap/ui/core/mvc/Controller",
  "sap/m/MessageBox"
], function (Controller, MessageBox) {
  "use strict";
  return Controller.extend("sap.training.exc.controller.App", {
    onSayHello: function () {
      MessageBox.information("Hello World");
    }
  });
});
⚠️
View controllers extend sap.ui.core.mvc.Controller. The lifecycle hook onInit runs once per view instance after controls are created — the right place for setup like binding a model.

Content Densities & Device Adaptation

Lesson 13

SAPUI5 supports Cozy (default for sap.m — larger, touch-friendly) and Compact (smaller, mouse-friendly) densities, triggered by CSS classes set high in the control tree (they cascade down).

CSS ClassDensity
sapUiSizeCozyLarger dimensions & spacing (touch)
sapUiSizeCompactSmaller dimensions & spacing (mouse)
 
getContentDensityClass: function () {
  if (!this._sContentDensityClass) {
    if (Device.support.touch) {
      this._sContentDensityClass = "sapUiSizeCozy";
    } else {
      this._sContentDensityClass = "sapUiSizeCompact";
    }
  }
  return this._sContentDensityClass;
}

// in the view controller's onInit:
this.getView().addStyleClass(this.getOwnerComponent().getContentDensityClass());
📋
The mandatory contentDensities object in the sap.ui5 namespace of manifest.json declares which densities the app supports. The FLP reads this and sets the body's density class automatically when running there.

Fragments — Reusable UI Parts & Dialogs

Lesson 15

Fragments solve the reuse problem without the overhead of a new control (dev effort) or a new view (runtime overhead). An XML fragment (*.fragment.xml) looks like a view but has no <View> root tag — instead <core:FragmentDefinition> if there's more than one root control.

↑ A fragment's content is inserted directly where it is placed — no HTML representation of its own

<core:FragmentDefinition xmlns="sap.m" xmlns:core="sap.ui.core">
  <Dialog id="dialog" title="Info" type="Message">
    <content>
      <Text text="Customer data is later saved via an OData service."/>
    </content>
    <beginButton>
      <Button text="Ok" press=".onCloseDialog"/>
    </beginButton>
  </Dialog>
</core:FragmentDefinition>
📎
In an XML view — <core:Fragment fragmentName="..." type="XML"/>, the view's controller becomes the fragment's controller automatically.
⚙️
Via API — this.loadFragment({name}) in a view controller, or sap.ui.core.Fragment.load() outside a controller.
onSave: function () {
  if (!this.pDialog) {
    this.pDialog = this.loadFragment({
      name: "sap.training.exc.view.Dialog"
    }).then(function (oDialog) {
      syncStyleClass(this.getOwnerComponent().getContentDensityClass(), this.getView(), oDialog);
      return oDialog;
    }.bind(this));
  }
  this.pDialog.then(function (oDialog) { oDialog.open(); });
}
⚠️
Dialogs render in the DOM's static area, outside the view — so the view's content-density class is not inherited automatically. Forward it manually with sap/ui/core/syncStyleClass.

JSON Model & Property Binding

Lesson 17

A model holds data and provides methods to get/set it. The JSON model (sap.ui.model.json.JSONModel) is the simplest, most common model type used in this course alongside the resource model and OData V4 model.

var oModel = new JSONModel();
this.getView().setModel(oModel, "customer");   // named model
oModel.setData({ CustomerName: "SAP SE", City: "Walldorf" });
oModel.loadData("model/data.json");             // GET + populate
var oData = oModel.getData();
🔤
Binding syntax: {path/to/data} for the default (unnamed) model, or {modelName>/path} for a named model. Absolute paths start with /; relative paths resolve against the control's binding context.
<Input value="{customer>/CustomerName}"/>
<Text text="{customer>/CustomerName}"/>
Binding ModeBehaviorJSON Model Default?
One-wayModel → View only
Two-wayModel ↔ View, both directions sync✅ default
One-timeView reads model once, no further updates

Aggregation Binding & Element Binding

Lessons 18–19

Besides property binding, SAPUI5 has two more binding types: aggregation binding (lists → tables) and element binding (one entity → detail view).

↑ items="{/Customers}" clones a template row (ColumnListItem) once per array element

<Table items="{/Customers}">
  <columns>...</columns>
  <items>
    <ColumnListItem>
      <cells>
        <Text text="{CustomerName}"/>   <!-- relative to array element -->
      </cells>
    </ColumnListItem>
  </items>
</Table>

↑ Selecting a customer sets the booking table's binding context via element binding

🔗
Element binding: a relative binding (e.g. items="{_Bookings}") has no data until it gets a context. On selectionChange, read oEvent.getParameter("listItem").getBindingContext() and call setBindingContext() on the detail table.
🔠

Data Types & Formatter Functions

Lessons 20–21

Both data types and formatter functions convert model data for display — but only data types can parse and validate user input (two-way); formatters are one-way only.

Simple TypeRepresentsNotes
sap.ui.model.type.IntegerWhole numberconstraints: minimum, maximum
sap.ui.model.type.DateDate, no timeformatOptions: pattern/style; constraints: min/max
sap.ui.model.type.CurrencyComposite: amount + currencyparts array; showMeasure hides the code

↑ All simple types inherit from sap.ui.model.SimpleType — custom types override formatValue/parseValue/validateValue

// Formatter module (formatter.js)
classText: function (sKey) {
  return sKey === "Y" ? "Economy Class" : sKey;
}

// XML view
<core:require="{formatter: 'sap/training/exc/model/formatter'}">
<Text text="{path:'Class', formatter:'formatter.classText'}"/>
⚠️
Using a formatter function on a binding automatically forces it to one-way — you cannot use a formatter in a two-way scenario. Invalid parse/validate results in a value that is not transferred to the model; use the Message Manager to surface errors on the UI.

Sorting, Filtering & Expression Binding

Lessons 22–23 
// Filter
var oFilter = new Filter("CustomerName", FilterOperator.Contains, "SAP");
this.byId("customerTable").getBinding("items").filter([oFilter]);

// Sort
var oSorter = new Sorter("City", true);  // true = descending

Filter data manuallysap.ui.model.Filter + binding.filter()
Sort data manuallysap.ui.model.Sorter + binding.sort()

Expression binding replaces trivial formatter functions inline: {= expr} (one-way, recalculates on model change) or {:= expr} (one-time). Embed model values with ${binding}. Requires compatVersion="edge".

OData V4 Model Instantiation & Bindings

Lesson 28

The OData V4 model (sap.ui.model.odata.v4.ODataModel) supports read, create, delete, two-way property updates, operation execution, batching, and server-side sort/filter.

"": {
  "dataSource": "mainService",
  "settings": {
    "synchronizationMode": "None",
    "operationMode": "Server",
    "earlyRequests": true,
    "autoExpandSelect": true
  }
}
ParameterPurpose
synchronizationModeMust be "None" — no other value supported
operationMode"Server" → sort/filter applied via $filter/$orderby URL params
earlyRequestsRequest $metadata/annotations/CSRF token as early as possible
autoExpandSelectAuto-generate $select/$expand — minimal payload, only UI-bound properties

↑ List binding (collection), Context binding (single entity), Property binding (primitive value)

🌐
Absolute bindings (start with /) trigger an OData request immediately. Relative bindings are initial — no data until they inherit a context from a parent list/context binding (same pattern as element binding with JSON models).

Generated Request Example

GET .../UX_Customer?$skip=0&$top=5&$orderby=City,CustomerName&$select=CustomerName,City,...

// Manual filter via API
oBinding.filter(new Filter("CustomerName", FilterOperator.Contains, "SAP"));
// → generates $filter=contains(CustomerName,'SAP')
⚠️
operationMode: "Server" is required for manual sort/filter via the binding API — without it, calling .sort()/.filter() raises an error.

Routing Configuration

Lesson 31

SAPUI5 uses hash-based navigation for single-page apps — no page reload, and hashes double as bookmarks/deep links. The routing config lives in sap.ui5/routing of manifest.json.

↑ config (defaults) → routes (name + pattern) → targets (view/component to display)

SectionPurpose
configGlobal defaults: viewType, viewPath, routerClass, controlId/controlAggregation, async
routesName + pattern (matched against the browser hash) + target(s)
targetsView/component to load + where (controlId/controlAggregation)
📱
routerClass: sap.m.routing.Router is optimized for mobile — it also triggers transitions (slide/flip/fade/show) during navigation, controlled via each target's viewLevel (navigation direction).
this.getRouter().initialize();   // in Component.js init()
⚠️
Without calling initialize() on the router in the component's init() method, the routing configuration is never activated — the current hash won't be evaluated.

    window.history.go(-1);
  } else {
    this.getOwnerComponent().getRouter().navTo("overview", {}, true);
  }
}
↩️
The browser's native Back button always uses browser history and can exit the app. A custom Back handler checks sap/ui/core/routing/History#getPreviousHash() — if there's no previous in-app hash, it navigates to a known route with bReplace=true instead of leaving the app.

Invalid Hashes & Mandatory Parameters

Lessons 34–35

↑ The bypassed property points to a notFound target shown for any unmatched hash

"routing": {
  "config": { "bypassed": { "target": "notFound" } },
  "targets": {
    "notFound": { "viewName": "NotFound", "transition": "show" }
  }
}

↑ Curly braces mark a mandatory parameter: pattern "detail/{customerId}"

// Sender
oRouter.navTo("detail", { customerId: "5" });

// Receiver (detail view controller, onInit)
this.getOwnerComponent().getRouter()
  .getRoute("detail")
  .attachPatternMatched(this._onObjectMatched, this);

_onObjectMatched: function (oEvent) {
  var sCustomerId = oEvent.getParameter("arguments").customerId;
}
⚠️
Mandatory parameters use {param} in the pattern — the router throws an error if navTo() is called without supplying them. attachPatternMatched's second argument rebinds this to the view controller inside the handler.

Q1: Which three sections make up the routing configuration in manifest.json?
✅ config, routes, targets
Q2: What method does the router provide to navigate to a specific route?
✅ navTo
Q3: What property specifies the fallback target shown when no route matches the current hash?
✅ bypassed
Q4: What must be called in Component.js init() so the current hash is evaluated automatically?
✅ this.getRouter().initialize();
Q5: For pattern "detail/{customerId}", which hashes match?
✅ Any hash supplying a value for customerId, e.g. detail/456 or detail/(00505604-...) — but not the bare "detail" or "detail/" with nothing after it

🎯 Unit 7 Key Takeaways

  • Hash-based navigation: no reload, supports bookmarks/deep links
  • Routing config = config (defaults) + routes (name/pattern) + targets (view/component)
  • Router must be initialized in Component.js init() via this.getRouter().initialize()
  • navTo(name, params, bReplace) — bReplace=true avoids a browser-history entry
  • Custom Back button: History.getPreviousHash() then window.history.go(-1) or navTo fallback
  • bypassed → notFound target for unmatched hashes; {param} = mandatory route parameter
